# Kaskad Taxi — VK Mini App

Полное руководство по развёртыванию и использованию новой системы такси.

---

## Архитектура

```
VK Bot (Group 3 DM)
  └── «Заказать такси» → отправляет ссылку на Mini App

Mini App (Next.js, /taxi)
  ├── ShiftGate      — проверка наличия водителей на смене
  ├── OrderFlow      — ник → попутчики → точка откуда → точка куда → промокод → оплата → подтвердить
  └── MapEditor      — только для роли RS: карта + зоны + точки + настройки

API Routes
  ├── GET  /api/taxi-user-role?vkUserId=...  — определяет роль и ник
  ├── GET  /api/taxi-config                  — возвращает карту, точки, зоны, настройки
  ├── POST /api/taxi-config                  — сохраняет изменения редактора
  ├── GET  /api/taxi-order-submit?action=shift-check — проверка смены
  └── POST /api/taxi-order-submit            — оформить заказ (сохранить + notify bot)

VK Bot
  └── pollPendingTaxiDispatch() — каждые 3 сек проверяет taxi_pending_dispatch.json,
      отправляет заказ в диспетчерскую (с фото карты) + уведомляет клиента в ЛС
```

## Быстрый старт — развернуть Mini App

### Шаг 1. Разверните Next.js проект на Vercel

```bash
# Через CLI
npm i -g vercel
vercel --prod

# Или через GitHub — подключите репозиторий в vercel.com/dashboard
```

После деплоя получите URL, например: `https://your-app.vercel.app`

### Шаг 2. Настройте переменную APP_URL в боте

В `scripts/long-polling.cjs` найдите строку:
```js
const APP_URL = process.env.APP_URL || 'https://your-app.vercel.app';
```
Замените на ваш реальный URL или задайте переменную окружения `APP_URL`.

### Шаг 3. Создайте VK Mini App

1. Перейдите на [vk.com/editapp?act=create](https://vk.com/editapp?act=create)
2. Выберите тип **Мини-приложение (VK Mini Apps)**
3. В настройках укажите URL вашего деплоя: `https://your-app.vercel.app/taxi`
4. Включите **VK Bridge** — библиотека автоматически передаёт `vk_user_id` через URL параметры
5. Сохраните App ID

### Шаг 4. Добавьте кнопку Mini App в VK группу

1. В настройках группы → «Приложения» → выберите ваше Mini App
2. Добавьте кнопку в боковую панель группы

## Конфигурация ролей

Роли определяются по `vk_user_id` через `scripts/data/staff.json`:

```json
{
  "123456789": {
    "nick": "PlayerNick",
    "role": "rs",
    "org": "taxi"
  }
}
```

| Роль | Описание | Поведение в Mini App |
|------|----------|----------------------|
| `rs` | Руководящий состав | При заказе — сначала проверка смены. Доступна кнопка «Карта» для редактирования |
| `ss` | Старший состав | Проверка смены при открытии, заказ такси |
| `staff` | Сотрудник | Проверка смены при открытии, заказ такси |
| `client` | Клиент (не в staff.json) | Проверка смены при открытии, только заказ |

## Редактор карты (RS only)

Доступен через кнопку «Карта» в шапке Mini App.

### Загрузка карты
- Нажмите **«Загрузить карту»** и выберите изображение (JPG/PNG)
- Карта сохраняется прямо в `taxi_points.json` как base64 (или внешний URL)

### Рисование зон
1. Нажмите **«Зона»** в тулбаре
2. Выберите тип: «Нельзя вызвать (откуда)», «Нельзя вызвать (куда)», «Запрещено полностью»
3. Кликайте по карте, добавляя вершины полигона (минимум 3)
4. Нажмите **«Завершить»**

Цветовое обозначение:
- **Красный** — нельзя вызвать такси отсюда
- **Жёлтый** — нельзя приехать сюда
- **Серый** — полный запрет

### Добавление точек
1. Нажмите **«Точка»** в тулбаре
2. Заполните название, выберите категорию и цену
3. Кликните по карте, чтобы установить координаты
4. Нажмите **«Добавить»**

### Настройки тарификации
- **Цена за пиксель** — основной коэффициент расчёта (рублей за единицу расстояния)
- **Множитель час-пик** — множитель в пиковое время (по умолчанию 1.3)
- **Час-пик с/до** — диапазон в МСК
- **Мин. цена** — нижний порог стоимости заказа

Не забудьте нажать **«Сохранить»** после всех изменений.

## Поток заказа в Mini App

```
Открытие приложения
  ↓
[Клиент/SS/Staff] ShiftGate — показывает водителей на смене
  ↓ (водители есть → «Заказать такси»)
[RS] ShiftGate — появляется только при нажатии «Заказать»
  ↓
Шаг 1. Никнейм
Шаг 2. Попутчики (до 2, необязательно)
Шаг 3. Откуда (категория → точка, точки в запрещённых зонах недоступны)
Шаг 4. Куда (аналогично, исключая выбранную «откуда»)
Шаг 5. Промокод (необязательно)
Шаг 6. Способ оплаты (наличные / счёт телефона +7% / банк +5%)
Шаг 7. Подтверждение → отправка
  ↓
POST /api/taxi-order-submit
  ↓
Bot poll → sendTaxiOrderToDispatch (диспетчерская получает: текст + фото карты с точками A и B)
Bot → sendMessage к клиенту в ЛС: «Заказ #N принят»
```

## Данные

| Файл | Содержимое |
|------|-----------|
| `scripts/data/taxi_points.json` | Категории, точки, зоны, настройки, base64 карты |
| `scripts/data/orders.json` | Все заказы (taxi + delivery) |
| `scripts/data/taxi_pending_dispatch.json` | Очередь для отправки боту в диспетчерскую |
| `scripts/data/staff.json` | Роли сотрудников по vk_user_id |
| `scripts/data/online_journal.json` | Кто на смене |

## Развёртывание бота (PM2)

```bash
# Установить зависимости
npm install

# Запустить бота
npx pm2 start ecosystem.config.cjs

# Или напрямую
node scripts/long-polling.cjs
```

Бот автоматически запустит `pollPendingTaxiDispatch()` и будет каждые 3 секунды проверять новые заказы из Mini App.
